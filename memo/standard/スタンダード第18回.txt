IDEで　設定→プラグイン→Lombok

configurations {
	compileOnly {
		extendsFrom annotationProcessor
	}
}

用途：コンパイル時に必要なライブラリ（アノテーション系）を JAR に含めず使えるようにする

よくある例

⦁	Lombok をプロジェクトに入れるとき

⦁	MapStruct などのコード生成系ライブラリ


compileOnly 'org.projectlombok:lombok'
annotationProcessor 'org.projectlombok:lombok'


compileOnly

⦁	Lombok は コンパイル時にだけ必要

⦁	JAR に含める必要はない（実行時には不要）

annotationProcessor

⦁	Lombok がコンパイル時に コードを自動生成 するために必要

⦁	これがないと @Getter, @Setter などのアノテーションが機能しない


IDEで　設定→ビルド、実行、デプロイ→コンパイラー→アノテーションプロセッサー→アノテーション処理を有効にする
